1. When prompted, indicate you would not like to test the installation media by pressing the right arrow key followed by Enter.
  2. Press Enter to pass the welcome screen.
  3. Select English as your language by pressing Enter.
  4. Select a "us" model keyboard by pressing Enter.
  5. Re-initialize the disk by pressing the right arrow key three times and pressing Enter.
  6. Accept the default time zone ("America/New York") by pressing Tab twice, followed by Enter.
  7. Select a password for the root account; we suggest using "simics" for your password.
  8. If you get a warning about using a weak password, use the right arrow key to highlight "Use Anyway" and press Enter.
  9. Use the up arrow key to highlight "Use entire drive," then press Tab twice to highlight "OK" and press Enter.
  10. Use the right arrow key to highlight "Write changes to disk" and press Enter.
  11. Wait as the system is installed.
  12. Press Enter to reboot.

The system will fully reboot. When it does so, we need to shut it down properly so that we can save the virtual state of the hard drive to the actual hard drive.

  1. Login using "root" as the user name.
  2. Enter your password ("simics" if you followed our recommendation).
  3. Type init 0 and press Enter.
  4. Wait for shutdown to complete.

After the virtual machine displays "Power down" switch to the host machine's terminal (where you originally entered the command ./simics scripts/install_centos67.simics). After it has twice printed the message, "Sleep state is unimplemented," interrupt simulation by pressing Ctrl+C.

You will now need to write the virtual disk's changes to your physical disk.

  1. At the simics> command prompt, enter the command save-persistent-state CENTOS.
  2. When the simics> command prompt returns, enter the command quit to exit Simics.
  3. From the normal Bash command line, enter the command mv CENTOS/viper.disk-... disk_images/centos67.craff
  4. You can now delete the remaining installation files by entering the command rm -rf CENTOS.
